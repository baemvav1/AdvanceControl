🔴 CRÍTICOS (RIESGO DE CRASH EN PRODUCCIÓN)

1. ConciliacionViewModel.cs:232-233
   ↳ Task.Result after Task.WhenAll()
   ↳ Patrón: await WhenAll() → acceso a .Result (anti-patrón UI)
   ↳ Riesgo: UI thread deadlock, freeze
   ↳ Fix: 5 min - Cambiar a: var estados = await estadosTask;

2. FacturaService.cs:63 & EstadoCuentaXmlService.cs:63
   ↳ Null coalescence en API deserialization
   ↳ Si API retorna null JSON → DTO fake con Success=true
   ↳ Riesgo: Confirmación falsa de operación fallida
   ↳ Fix: 15 min - Validar null antes de coalescence

3. DetailFacturaViewModel.cs:113 (similar en línea 114, 329)
   ↳ Valida Factura!=null pero NO Conceptos/Abonos
   ↳ Riesgo: NullReferenceException en línea 121
   ↳ Fix: 5 min - Añadir validación de todas las propiedades

4. ReemplazarColeccion (4 ubicaciones: 223, 275, 520, 602)
   ↳ Método sin null check en origen
   ↳ Riesgo: foreach sobre null → crash
   ↳ Impacto: MÁXIMO (usado en TODO flujo de detalle)
   ↳ Fix: 10 min - Proteger con if (origen != null)

5. NavigationService.cs:40
   ↳ Empty catch block sin logging
   ↳ Riesgo: Errores de navegación invisibles
   ↳ Fix: 5 min - Añadir Debug.WriteLine()

6. MainViewModel.cs:96-107
   ↳ Fire-and-forget Task en constructor
   ↳ Riesgo: Exception en background thread no propagada
   ↳ Fix: 10 min - Mover a OnNavigatedTo o hacer observable
